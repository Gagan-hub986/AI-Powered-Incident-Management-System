# AI-Powered Incident Management System

## Overview

This project is an AI-powered Incident Management solution built using UiPath Maestro, RPA Workflows, AI Agent, Orchestrator APIs, Zoho Desk APIs, and Outlook Event Triggers.

The solution automatically monitors the latest completed Orchestrator job, analyzes its execution status and logs, identifies failures, raises Zoho Desk tickets when required, notifies the responsible developer, and maintains a centralized execution history.

---

# Project Structure

The ZIP folder contains the following files:

## 1. Incident Management Bot Data and Config.uis

This RPA workflow performs the initial configuration.

Responsibilities:

* Reads the Config Excel file from OneDrive.
* Reads the Bot Information Excel.
* Extracts all configuration values.
* Reads the AI Agent knowledge base.
* Passes the required configuration, Bot Data, and DataTables to the next workflow.

---

## 2. Incident Management Orchestrator Data.uis

This workflow communicates with UiPath Orchestrator.

Responsibilities:

* Retrieves the Orchestrator Personal Access Token from an Asset.
* Calls the Orchestrator API to fetch the latest completed job.
* Retrieves complete execution details.
* Uses the Job Key to fetch execution logs.
* Stores Job Details and Logs in JSON format.
* Passes the information to the AI Agent.

---

## 3. Incident Management Orchestrator Agent.uis

This is the AI Agent and acts as the brain of the entire system.

Responsibilities:

* Reads Job Details.
* Reads Execution Logs.
* Reads Bot Knowledge Base.
* Determines whether the execution is actually successful or failed.
* Finds the exact failure reason.
* Identifies the root cause.
* Suggests the possible resolution.
* Estimates the expected resolution time.
* Determines ticket priority.
* Identifies the responsible developer.
* Passes the decision to the next workflow.

---

## 4. Incident Management Zoho Desk.uis

Executed only when the AI Agent identifies a failed execution.

Responsibilities:

* Creates a Zoho Desk ticket.
* Uses all information generated by the AI Agent.
* Returns the Ticket ID and Status.

---

## 5. Incident Management Records.uis

Maintains the centralized execution history.

Responsibilities:

* Reads the Execution History Excel.
* Creates a new entry for every execution.
* Stores:

  * Process Name
  * Job Status
  * Failure Reason
  * Ticket Number
  * Ticket Status
  * Responsible Owner
  * Timestamp
  * Other execution details

Every execution (Pass or Fail) is recorded.

---

## 6. Incident Management Flow (Maestro BPMN)

This is the main Maestro workflow.

Responsibilities:

* Connects all RPA workflows.
* Starts automatically through an Outlook Event Trigger.
* Controls the complete end-to-end process.

Workflow:

Email Received

↓

Read Config

↓

Fetch Orchestrator Job Data

↓

AI Agent

↓

Decision

↓

If Success → Update Records

↓

If Failure

↓

Create Zoho Ticket

↓

Send Notification Email

↓

Update Records

↓

End

# Orchestrator Configuration

## Generate Personal Access Token

1. Open UiPath Cloud.
2. Open Orchestrator.
3. Click your profile icon.
4. Select **Preferences**.
5. Open **Personal Access Token**.
6. Click **Create Token**.
7. Copy the generated token.

---

## Create Orchestrator Asset

Create a Text Asset named:

**Orchestrator_token**

Paste the generated Personal Access Token into the Asset in the same folder where your Incident Mangement Bot files are there.

---

## Verify Asset Usage

Open:

**Incident Management Orchestrator Data flow**

Ensure the **Get Asset** activity is retrieving:

**Orchestrator_token**

from the correct folder.

# Finding Folder ID

Open the desired Orchestrator Folder.

The browser URL contains:

fid=XXXXXXXX

The value after **fid=** is the Folder ID.

Copy this value into Config.xlsx.

---

# Event Trigger Configuration

The Maestro BPMN uses an Outlook Event Trigger.

When published, the Event Trigger is created automatically.

If you want to monitor a different mailbox or folder:

1. Open the Maestro BPMN.
2. Select the **Email Received** Event.
3. Click the **Tools** icon, on the right side.
4. Select your Outlook account.
5. Choose the folder you want to monitor.
6. Publish the workflow again.

Whenever a new email arrives in the selected folder, the workflow starts automatically.

# Notes

* Always publish the latest Maestro BPMN after making changes.
* Replace all connections with your own before running the solution.
* Ensure the Personal Access Token Asset exists.
* Verify the Folder ID in Config.xlsx.
* Upload both Excel files to OneDrive before execution.
* Every execution is recorded for complete transparency.
* Make sure agent priority is set to highest in Orchestrator
* Make a different folder for Incident Management Flows. Dont Mix it in your real automation folder to avoid loop.
